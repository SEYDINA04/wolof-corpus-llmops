📊 ÉTAPES EXPLIQUÉES:

✅ ÉTAPE 1: CHUNKING
   Audio 5,211s → 473 chunks de 20s (overlap 2s)
   Pourquoi: Plus de contexte syntaxe Wolof

✅ ÉTAPE 2: CONFIDENCE EXTRACTION
   Par chunk: score 0-1.0 = certitude du modèle
   Log-probs des tokens générés
   Filtre bas confidence = hallucinations

✅ ÉTAPE 3: POST-PROCESSING
   - Dédupliquer: "mot mot mot" → "mot" (>3 répétitions)
   - Filtrer: chunks avec confidence < threshold
   - Fusionner: détecter overlap, garder 1×
   
✅ ÉTAPE 4: WER/CER MEASUREMENT
   WER = pourcentage mots mal transcrits
   CER = pourcentage caractères mal transcrits
   Besoin référence Wolof pour calculer

❌ PROBLÈMES COURANTS:

"ModuleNotFoundError: whosper"
→ Relancer Cellule 2 (Installation)
→ Attendre ~30s

"CUDA out of memory"
→ Réduire CHUNK_DURATION à 15
→ Ou utiliser CPU (plus lent)

Confidence trop basse (< 0.4 moyenne)
→ Augmenter overlap: STRIDE = 15
→ Vérifier qualité audio (bruitée?)
→ Whosper peut ne pas être optimisé pour ce dialecte

JSON non exporté
→ Vérifier OUTPUT_DIR existe
→ Check Colab Files panel (à droite)

📈 ÉVALUATION RÉSULTATS:

CONFIANCE MOYENNE:
  > 0.7 = Excellente
  0.6-0.7 = Bonne
  0.5-0.6 = Acceptable
  < 0.5 = Faible (relancer avec overlap ++)

WER (Word Error Rate):
  < 15% = Excellent (rare)
  15-25% = Très bon
  25-35% = Bon (acceptable)
  35-45% = Moyen
  > 45% = Faible (besoin fine-tune)

CER (Char Error Rate):
  < 10% = Excellent
  10-15% = Très bon
  15-20% = Bon
